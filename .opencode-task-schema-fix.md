## Files to Modify

### 1. `schema/schema.sql` — Apply these column changes:

#### A. Remove `event_time` from `health_entries` (redundant with `timestamp`)
- Remove the `event_time TIMESTAMPTZ` column definition
- Remove any DEFAULT or constraint on `event_time`
- Update ALL indexes, functions, and RLS policies that reference `event_time` to use `timestamp` instead
- Update `search_health_entries` RPC function if it references `event_time`

#### B. Remove `ingestion_source` from `health_entries` (redundant with `source`)
- Remove the `ingestion_source` column definition
- Remove any DEFAULT on `ingestion_source`
- Update anything that references `ingestion_source` to use `source` instead

#### C. Remove `dates` column from `memories` table
- Remove `dates TEXT[]` column

#### D. Remove `decisions` column from `projects` table
- Remove `decisions JSONB` column

#### E. Remove `content_fingerprint` column from `memories` table
- It's superseded by the `external_id` upsert pattern

#### F. Remove `aliases` column from `entities` table (never populated)
- Remove `aliases TEXT[]` column

#### G. Add `numeric_value NUMERIC` column to `training_logs`
- Add: `numeric_value NUMERIC,`
- Place it near other value columns like `volume_kg`, `rpe`

#### H. Add indexes on `user_id` FK columns
- `CREATE INDEX idx_memories_user_id ON memories(user_id);`
- `CREATE INDEX idx_projects_user_id ON projects(user_id);`
- `CREATE INDEX idx_health_entries_user_id ON health_entries(user_id);`
- `CREATE INDEX idx_training_logs_user_id ON training_logs(user_id);`
