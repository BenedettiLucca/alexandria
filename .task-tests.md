## Structure

### 1. server/index.test.ts — Deno tests for the MCP server

Create a test file that tests:

- queryTable — builds and executes Supabase queries with filters
- simpleClassify — keyword-based classification (no API)
- recordToText — converts health records to text

- sanitizeClassification — validates LLM output, falls back to defaults
- extractNumericValue — extracts primary numeric from value JSONB

b) MCP tool input validation:
- For each of the 25 tools, test that valid inputs are accepted and invalid inputs are rejected (wrong types, missing required fields, out-of-range values like importance=15)

c) Auth middleware:
- Test that requests without auth return 401
- Test that x-brain-key with wrong value returns 401

d) Classification logic:
- simpleClassify returns correct categories for: workout keywords, bug keywords, idea keywords, decision keywords, recipe keywords, travel keywords, purchase keywords, people keywords, default

To mock Supabase, create mock functions that return controlled data. Do NOT try to import the actual server module (it has side effects with Deno.serve). Instead, extract the pure functions for testing by copying/adapting them, or test via the tool handler signatures directly.

The deno.json already has import maps. Add test deps if needed to server/deno.json.
